TCP Theory

1. Introduction to TCP

Transmission Control Protocol (TCP) is a connection-oriented protocol used for reliable data transmission. TCP ensures data is delivered in order, without errors, and without loss. It is used in applications like HTTP, FTP, Email, and File Transfer.

3. TCP Header

TCP header contains important fields such as Source Port, Destination Port, Sequence Number, Acknowledgement Number, Flags, Window Size, and Checksum.

4. Sequence Number and Acknowledgement Number

Sequence Number identifies the byte number of the data being sent. Acknowledgement Number indicates the next expected byte from the sender. These numbers ensure reliable and ordered data delivery.

5. TCP Flags

  • SYN – Synchronize sequence numbers
  • ACK – Acknowledgement
  • FIN – Finish connection
  • RST – Reset connection
  • PSH – Push data immediately
  • URG – Urgent data

6. TCP 3-Way Handshake

TCP uses a 3-way handshake to establish a connection between client and server.

  1. Client sends SYN
  2. Server sends SYN-ACK
  3. Client sends ACK

7. Connection Termination (4-Way Handshake)

  1. Client sends FIN
  2. Server sends ACK
  3. Server sends FIN
  4. Client sends ACK

8. TCP State Machine

TCP connection goes through multiple states such as CLOSED, SYN_SENT, SYN_RECEIVED, ESTABLISHED, FIN_WAIT, CLOSE_WAIT, LAST_ACK, and TIME_WAIT.

9. Flow Control (Sliding Window)

TCP uses sliding window mechanism for flow control. It ensures that sender does not send more data than receiver can handle.

10. Congestion Control

TCP uses congestion control algorithms like Slow Start, Congestion Avoidance, Fast Retransmit, and Fast Recovery to control network congestion.
