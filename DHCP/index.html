DHCP Theory

Everything you need to know — from basics to exam-level depth

What is DHCP?

Dynamic Host Configuration Protocol (DHCP) is a client-server protocol that automatically assigns IP addresses and other network configuration parameters to devices on a network.

Why DHCP is Needed

  • Manual IP configuration is time-consuming and error-prone
  • Duplicate IPs cause network conflicts
  • Not scalable for large or dynamic networks
💡 DHCP automates everything → plug-and-play networking for any device that joins the network

What DHCP Provides

When a client connects, DHCP automatically configures:

🔢 IP AddressUnique network identifier for the device
🔲 Subnet MaskDefines the network boundary
🚪 Default GatewayRouter address to reach other networks
🌐 DNS ServerResolves domain names to IPs

⭐ The DORA Process

DORA is the four-step handshake that DHCP uses to assign an IP address. Every time a device connects to a network, it performs DORA.

D

1. DISCOVER

Client has no IP yet. It broadcasts a DHCPDISCOVER message to locate any DHCP servers on the network.

Source IP: 0.0.0.0 Destination: 255.255.255.255 Protocol: UDP
O

2. OFFER

The DHCP server receives the discover, picks an available IP from its pool, and sends a DHCPOFFER unicast back to the client.

Offered IP address Subnet mask & gateway Lease duration
R

3. REQUEST

Client accepts the offer and broadcasts a DHCPREQUEST to confirm it wants that specific IP. Broadcast allows other servers to know the offer was accepted.

Client selects one server Still broadcast (notifies all servers) Contains offered IP address
A

4. ACK

Server sends DHCPACK confirming the IP assignment. The client now configures its network interface and the lease timer begins.

IP lease begins Countdown timer starts Client is now BOUND

Lease Mechanism

IP addresses are not permanently assigned — they are given for a lease period.

  • At 50% of lease: client sends unicast DHCPREQUEST to renew
  • At 87.5% of lease: client broadcasts DHCPREQUEST if no response
  • At 100% (expiry): IP is released back to the pool, client must restart DORA

APIPA (Automatic Private IP Addressing)

⚠️ If DHCP fails or no server is reachable, the operating system auto-assigns an IP from the 169.254.x.x range. This is only usable within the same local subnet — no internet access.

✅ Advantages

  • Automatic IP configuration — no manual entry needed
  • Reduces administrative workload drastically
  • Efficient IP reuse: IPs return to pool after lease expires
  • Centralized management of all network configurations
  • Scales to networks of any size

❌ Limitations

  • Single point of failure — if server goes down, no new IPs can be assigned
  • Dynamic IPs can break active connections when lease changes
  • Not recommended for servers (use static IPs for servers)
  • Vulnerable to DHCP spoofing / starvation attacks
