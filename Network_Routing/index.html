Adaptive Network Routing & Traffic Analysis

An interactive simulator for Dijkstra's shortest-path algorithm, end-to-end delay modelling, and multi-path performance analysis across a realistic ISP-topology network.

Project Overview

This simulator demonstrates how packets are routed across a network of 8 routers using Dijkstra's algorithm — the foundation of OSPF. It models all four delay components, computes routing tables, visualises the shortest path, and lets you explore alternative paths and algorithm comparisons.

Routing Theory Summary

  • Dijkstra (Link-State) runs in O((V+E) log V) with a min-heap.
  • Link cost = propagation + transmission + processing delay.
  • OSPF floods Link-State Advertisements (LSAs) so all routers hold identical topology maps.
  • ECMP: multiple equal-cost paths allow load-balancing across links.
  • Bandwidth-Delay Product determines optimal TCP window size.
  • QoS grades follow ITU-T G.114 one-way delay thresholds.

Routing Theory

1. Introduction to Routing

Routing is the Network Layer (L3) process of selecting paths across a network for traffic. A router maintains a Routing Information Base (RIB) built by protocols such as OSPF or RIP and uses it to forward packets hop-by-hop toward their destination.

2. Dijkstra's Algorithm (Link-State)

Dijkstra's algorithm computes the single-source shortest path tree. It is the mathematical engine inside OSPF.

PropertyValue
Time ComplexityO((V + E) log V) with binary min-heap
Space ComplexityO(V) for distance & predecessor arrays
InputWeighted undirected graph G(V, E)
OutputShortest-path tree rooted at source
Routing ProtocolOSPF (Open Shortest Path First)

Steps:

  1. Initialise dist[src] = 0; dist[all others] = ∞
  2. Push source into a min-heap (priority queue)
  3. Pop minimum-distance node u; mark settled
  4. For each neighbour v: if dist[u] + w(u,v) < dist[v] → relax
  5. Repeat until all nodes settled
  6. Trace predecessor map to reconstruct path

Algorithm Comparison

🔵 Dijkstra (Link-State)

  • Used by: OSPF, IS-IS
  • Topology: Full map via LSA flooding
  • Complexity: O((V+E) log V)
  • Convergence: Fast (~ms after LSA)
  • Memory: O(V²) — full graph stored
  • Loop-free: Yes (SPT guarantees)
  • Best for: Large enterprise / ISP cores
  • Metric: Configurable link cost

🔴 Bellman-Ford (Distance-Vector)

  • Used by: RIP, BGP (path-vector variant)
  • Topology: Partial — only neighbour info
  • Complexity: O(V × E)
  • Convergence: Slow (count-to-infinity)
  • Memory: O(V) — distance table only
  • Loop risk: Yes (requires split-horizon)
  • Best for: Small / stub networks
  • Metric: Hop count (RIP) or path attributes (BGP)
