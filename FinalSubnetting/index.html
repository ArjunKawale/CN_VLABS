Interactive simulator for understanding IP addressing, subnet masks, CIDR notation, and network subnetting.

Project Overview

Subnetting is the process of dividing a large network into smaller, more efficient sub-networks. This simulator demonstrates binary conversion, subnet ID calculation, subnetting exercises, and dynamic practice problems to build your networking skills.

Subnetting Theory Summary

  • An IP address is a unique 32-bit identifier for network devices.
  • Subnet masks determine network and host portions of an IP.
  • CIDR notation provides a compact way to specify subnet masks.
  • Subnetting borrows bits from the host portion to create sub-networks.
  • Each subnet has a network address and a broadcast address.
  • IP addresses are classified into Class A, B, C, D, and E.

Subnetting Theory

1. What is an IP Address?

An IP (Internet Protocol) address is a unique 32-bit identifier assigned to every device on a network. It is used for communication between devices in a network. IP addresses are written in dotted decimal notation, e.g., 192.168.1.0.

2. Dotted Decimal Notation

A 32-bit IP address is split into four 8-bit octets. Each octet is converted to a decimal number separated by periods. For example: 11000000.10101000.00000001.00000000 = 192.168.1.0

3. Network ID and Host ID

An IP address consists of a Network ID (identifies the network) and a Host ID (identifies the specific device). The subnet mask determines which part is the network ID and which is the host ID.

4. Subnet Mask

A subnet mask is a 32-bit number that separates the IP address into the network and host portions. For example, a subnet mask of 255.255.255.0 means the first 24 bits are the network ID and the remaining 8 bits are for hosts.

5. CIDR Notation

CIDR (Classless Inter-Domain Routing) notation represents the subnet mask as a suffix. For example, /24 means the first 24 bits are the network portion. This is equivalent to 255.255.255.0.

7. Subnetting

Subnetting is the process of dividing a large network into smaller sub-networks (subnets). This is done by borrowing bits from the host portion of the IP address. Subnetting improves network efficiency, security, and management.
